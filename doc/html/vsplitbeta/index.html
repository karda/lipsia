LIPSIA     Split GLM result (the betas) of a single subject scan into separate files.
vsplitbeta

This program splits the GLM result (the betas) of a single subject scan into separate files. This is part of the general single-subject-pipeline for our data mining approach and cannot be used without a single-subject data design as generated by vsvmdesign.pl


vsplitbeta -in betas.v -base base_name -conversion conversion.txt


Examples for calling 'vsplitbeta'

vsvmdesign.pl design.txt -out_design design_single.txt -out_conversion conversion.txt -aggregate 2

vgendesign -in design_single.txt -out design_single.v -tr 1 -ntimesteps 1200 -deriv 0

vcolorglm -in ntbsm_data.v -design design_single.v -out beta_single.v -minval 200

vsplitbeta -in beta_single.v -base "separate_betas" -conversion conversion.txt

This example goes through the steps to split the scan of a single subject into separate “single subject samples” by treating every trial within the scan as a separate sample. The program vsvmdesign.pl generates a new design file that defines the “single subject samples” and with vgendesign and vcolorglm the separate samples are calculated. The program vsplitbeta then splits up the output of vcolorglm into separate files that are usable by other statistical processing algorithms.


Parameters of 'vsplitbeta':
General Options:
-in betas.v
Input file. Required
-base base_name
Basename of extracted images. Required.
-conversion conversion.txt
Name of conversion file. Required.
