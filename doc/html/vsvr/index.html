LIPSIA     Two Group Support Vector Machine Classification
vsvr

This program performs a support vector regression analysis on one group of contrast images where each image is associated with a dependent or response variable (e.g. age, IQ, level of enlightenment). The output is a 3D map of the weights of the classifier as well as p-values of requested.

Input data consists of a set of vista files and a text file containing the response values of each sample. Each vista file corresponds to one sample and contains exactly one image with 3D data of the brain. The 3D data can be derived from any imaginable pipeline, i.e. beta values from GLM or centrality values from ECM. The response text file contains one sample response per line with a period as the decimal separator and no thousands separator.


vsvr -in samples.v -y response.txt [-scale] [-pca] [-saveperm] [-nperm number of permutations] [-j nprocs] [svm options]


Examples for calling 'vsvr'

Example 1:

vsvr -in data_*.v -y resonse.txt -out data_svr.v -scale

In this example we calculate a simple support vector regression on two groups of images containing contrast images, the data gets scaled in the process, no p-values are calculated

Example 2:

vsvr -in data_*.v -y resonse.txt -out data_svr.v -pca -nperm 10000 -scale

In this example we calculate a simple support vector regression on two groups of images containing contrast images, the data gets scaled and piped through a principal component analysis in the process. Finally, p-values are calculated based on a permutation algorithm.


Parameters of 'vsvr':
General Options:
-in samples*.v
Input files, required
-y response.txt
Response values, required
-scale
Whether to scale data (default: false)
-pca
Wheter to process data with a principal component analysis (default: false)
-nperm number of permutations
Whether to calculate permutations and therefore p-values, number of permutations (default: 0 = do not permutate)
p-values are calculated with a non-parametric permutation test: For each voxel, a null distribution is established by calculating support vector regressions for random permutations of the original problem. A random permutation of the original problem is the random assignation of samples to responses. The p-value of the real weight of a voxel is then the probabilty that any randomly calculated weight is either higher or smaller (two-tailed) then the real weight.
Consequently, the granularity of p-values depends on the number of calculated permutations \(n\). Possible p-values are \(\frac{2 * k}{n}, k \in [0,\frac{n}{2}] \), e.g. \(0, \frac{2}{n}, \frac{4}{n},... \).
-saveperm
Whether to also write permutations to output file (default: false)
-j nprocs
number of processors to use, '0' to use all (default: 4)
SVM Options
-svm_cache_size cache_size
cache size parameter (in MByte; default: 100)
-svm_p p
p parameter (default: 0.1)
-svm_C C
C parameter (default: 1.0)
