LIPSIA     Generate a design file for “single subject samples”
vsvmdesign.pl

The program takes a classical design file that specifies the trials in a block design experiment and generates a design file that separates each trial (or an aggregation of trials). For example, if a scan is a sequence of trials A, B, A, B, A, B, A, B and we want to separate trials, the new design file will specify trials A1, B1, A2, B2, A3, B3, A4, B4. This new design file is helpful to run a GLM analysis on the subject scan to get one beta per trial.

If aggregation is desired, two or more trials are aggregated to one sample-trial, e.g. for the previous example and an aggregation of two trials, the new design file will specify: A1, B1, A1, B1, A2, B2, A2, B2.

The output of the program is the new design file and a conversion file that contains the mapping from the new trial to the original classification (e.g. for the previous example this would be A1 → A, A2 → A, A3 → A, B1 → B, B2 → B, B3 → B).


vsvmdesign.pl [-out_design new_design_file] [-out_conversion conversion_file] [-aggregate aggregate] design_file


Example for calling 'vsvmdesign.pl'

Example 1

vsvmdesign.pl -out_design design_single.txt -out_conversion conversion.txt -aggregate 4 design.txt

The example takes the design file design.txt and generates design_single.txt and conversion.txt where it aggregates two trials to one sample

Example 2:

vsvmdesign.pl design.txt -out_design design_single.txt -out_conversion conversion.txt

vgendesign -in design_single.txt -out design_single.v -tr 1 -ntimesteps 1200 -deriv 0

vcolorglm -in ntbsm_data.v -design design_single.v -out beta_single.v -minval 200

This example is the full pipeline for conducting a GLM on a single subject. We generate a design that separates the trials and then we run a GLM analysis on the subject scan. The output file beta_single.v contains as many images as there were trials in the original scan.


Parameters of 'vsvmdesign.pl':
General Options:
-out_design new_design_file
Name of the output design file (default: design_single.txt)
-out_conversion conversion_file
Name of the output conversion file (default: conversion.txt)
-aggregate aggregate
How many trials to aggregate (default: 1)
